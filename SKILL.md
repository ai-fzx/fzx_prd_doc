# PRD Writer - 产品需求文档技能

> 用于快速编写专业的产品需求文档（Product Requirements Document）

## 触发词

- 写PRD、写产品需求文档、需求文档、PRD文档
- 产品文档、产品说明、需求说明
- 做一个产品、规划产品

## 一、背景与目标

### 1.1 背景说明

> 为什么要做这个产品？解决什么痛点？

- 描述当前存在的问题/痛点
- 目标用户是谁
- 市场规模（如果有）

> 通过这个产品要达到什么效果？

- 核心目标（1-2个）

## 使用说明

### 快速生成PRD

```
请帮我写一份[产品名称]的PRD，核心功能是[功能描述]，目标用户是[用户群体]。
```

### 按模块补充

```
请帮我完善[具体模块]，例如：用户需求、技术设计、页面设计等。
```

### 检查完整性

```
请检查当前PRD是否包含所有必要模块，帮我补充缺失部分。
```

---

## 质量检查清单

完成PRD后，检查以下项：

- [ ] 有清晰的背景说明
- [ ] 有具体的用户画像
- [ ] 需求有明确优先级
- [ ] 核心功能有详细流程
- [ ] 关键页面有交互说明
- [ ] 技术方案可行
- [ ] 有明确的发布计划
- [ ] 有验收标准

## 示例

### 示例1：简单产品

```markdown
# 打卡小程序 PRD

## 一、背景

因为很多人想要养成早起习惯，但缺乏动力。做一个早起打卡小程序，帮助用户早起并获得成就感。

## 二、目标

- 核心：帮助用户早起打卡并坚持
- 指标：7日留存>40%

## 三、功能

| 功能 | 描述 |
|------|------|
| 打卡 | 每天6-9点打卡 |
| 排行榜 | 好友排行 |
| 奖励 | 连续打卡奖励 |
```
